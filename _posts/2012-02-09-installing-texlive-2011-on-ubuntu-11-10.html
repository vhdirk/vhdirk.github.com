The vanilla TexLive comes with a package management tool called tlmgr which is particularly usefull when you want to install additional packages. The TexLive version packaged in ubuntu does not have this tool since it would conflict with apt.

Anyway, installing Tex packages the ubuntu way is a bit annoying, so my preferred solution is to install the vanilla version, and a fake package that replaces the ubuntu packages. It might even contain some awesome new features (or bugs); ubuntu's version is already 3 years old.

So, here it goes:
  1. Ensure that the only Debian TeX Live packages installed are tex-common, texinfo, and perhaps lmodern
  2. Install vanilla TeX Live as root, system-wide: http://www.tug.org/texlive/quickinstall.html
    During install, enter L to enable symlinks and hit 3 times “Enter” to keep default settings.
    note: you can also add the binary path of the installation to /etc/environment
  3. Tell APT about your TeX Live installation by building a dummy package using equivs:

    sudo aptitude install equivs
    mkdir /tmp/tl-equivs && cd /tmp/tl-equivs
    equivs-control texlive-local
    # edit texlive-local (see example)
    equivs-build texlive-local
    sudo dpkg -i texlive-local_*.deb
  4. Voila, all done.
